\chapter{From Structured Inference to Learned Refinement Dynamics}\label{chapter:4_denoising_and_mrf_inference}
\chaptermark{Refinement Dynamics}

\section{Introduction}
\label{sec:chapter_4_intro}

The previous chapter introduced MRF-based structured prediction models for BILPs. In that setting, a variable--constraint graph was mapped to MRF factor parameters, approximate inference produced unary marginals, and feasibility-preserving decoders converted these scores into binary assignments.

This chapter keeps the same separation between prediction, structured inference, and decoding, but changes how the prediction signal is produced. Instead of obtaining one score vector from a single forward pass, we study learned refinement dynamics over candidate solution states. A candidate state is represented as a vector in $[0,1]^n$, interpreted as Bernoulli parameters for the binary decision variables. The model is conditioned on the BILP instance, the current state, and a refinement time or noise level.

The motivation comes from the generative and denoising viewpoint introduced in Section~\ref{sec:generative_denoising}. Here, however, the generated object is not evaluated as an unconstrained sample. It is used as a score signal for the same feasibility-preserving decoders used in the previous chapter. The question is therefore whether learned refinement produces better decoding scores than one-shot prediction.

We study two refinement objectives. The first is flow-based: it defines a linear path between a random Bernoulli-parameter source state and a reference solution, and trains the model either to predict the velocity along this path or to predict the clean endpoint. The second is diffusion-inspired: it corrupts reference solutions by replacement noise and trains the model to reconstruct the clean assignment. In both cases, the new feature is state conditioning: the MRF parameters depend not only on the instance, but also on the current candidate state.

The test-time budget also changes. In the previous chapter, the main inference budget came from approximate inference and repeated GD updates. Here, the learned model itself is iterative, so the number of refinement steps becomes an additional runtime--quality parameter.

The experiments use Set Cover (SC) and Maximum Independent Set (MIS), as before. These two families are useful because their decoding errors behave differently. In SC, adding sets can repair uncovered elements. In MIS, selecting one vertex immediately excludes its neighbors, making early ranking errors harder to recover from.

The contributions of this chapter are:
\begin{enumerate}
    \item a state-conditioned MRF refinement architecture in which factor parameters depend on the instance, the current Bernoulli-parameter state, and the refinement time;

    \item a flow-based refinement objective for BILP solution prediction, using both velocity prediction and direct endpoint prediction;

    \item a diffusion-inspired denoising objective based on replacement corruption of reference solutions;

    \item a unified decoding protocol that evaluates refined states with the same feasibility-preserving decoders as the previous chapter;

    \item an empirical comparison of direct, LBP-based, and NMF-based refinement models on SC and MIS, including the effect of unrolling and backend reinference.
\end{enumerate}

\section{State-Conditioned MRF Refinement Models}
\label{sec:state_conditioned_mrf_refinement}

We first adapt the denoising and flow-matching framework of Section~\ref{sec:generative_denoising} to BILP solution prediction. The state space is the relaxed hypercube $[0,1]^n$, whose coordinates are interpreted as Bernoulli parameters for the binary decision variables. Unlike the predictors of the previous chapter, the MRF parameters are now conditioned on the current state of the refinement process. The graph encoder therefore predicts factor parameters for an MRF that changes along the refinement trajectory.

\subsection{Refinement states and probability path}

Let $\PROB$ be a BILP instance with $n$ binary decision variables, and let $\FMPREF\in\{0,1\}^n$ denote a reference solution. We represent an intermediate candidate assignment by a relaxed vector
\begin{equation*}
    \FMP \in [0,1]^n,
\end{equation*}
where $\FMP_i$ is interpreted as the current relaxed score or probability that variable $i$ should be set to one.

The source state $\FMSOURCESTATE$ is sampled componentwise from an uninformative relaxed distribution:
\begin{equation}
    \FMSOURCESTATE_i \sim \mathcal{U}(0,1),
    \qquad i=1,\ldots,n.
    \label{eq:flow_source_distribution}
\end{equation}
This provides an initial point in $[0,1]^n$ with no instance-specific information.

Given $\FMSOURCESTATE$, the reference solution $\FMPREF$, and a time $t\in[0,1]$, we use the linear probability path
\begin{equation}
    \FMP_t = (1-t)\FMSOURCESTATE + t \FMPREF .
    \label{eq:flow_probability_path}
\end{equation}
Thus $\FMP_0=\FMSOURCESTATE$ and $\FMP_1=\FMPREF$. Intermediate states $\FMP_t$ are relaxed assignments in $[0,1]^n$. This is the binary-solution analogue of the interpolation path introduced in Section~\ref{sec:generative_denoising}.

\subsection{State-conditioned MRF architecture}
\label{sec:state_conditioned_mrf_architecture}

The architecture reuses the structured prediction pipeline from Chapter~\ref{chapter:3_differentiable_mrf_inference_for_BILPs}, with one additional input: the current refinement state. The BILP instance is represented by the fixed variable--constraint graph $\VCGFULL$. As in the graph-feature representation described in Section~\ref{sec:neural_networks}, the graph structure specifies the message-passing topology, while node and edge features carry the numerical information.

At time $t$, the graph structure is unchanged. We condition the model through the time-dependent feature collection
\begin{equation}
    \FMFEATURES_t
    =
    \left(
    \{\FEATURES^V_t(i)\}_{i\in\VCGV},
    \{\FEATURES^C_t(j)\}_{j\in\VCGC},
    \{\FEATURES^E(i,j)\}_{(i,j)\in\VCGE}
    \right),
\end{equation}
with
\begin{equation}
    \FEATURES^V_t(i) = [\FEATURES^V(i), \FMP_{t,i}, \FMTIMEEMB(t)],
    \qquad
    \FEATURES^C_t(j) = [\FEATURES^C(j), 0, \FMTIMEEMB(t)].
    \label{eq:flow_state_conditioned_features}
\end{equation}
The variable features therefore contain the current relaxed value of each decision variable, while the constraint features receive a zero placeholder in the same coordinate. Both variable and constraint nodes receive the same time embedding.

The graph encoder processes $(\VCG,\FMFEATURES_t)$ and produces node embeddings. These embeddings are mapped to MRF factor parameters. As in Chapter~\ref{chapter:3_differentiable_mrf_inference_for_BILPs}, unary factors are associated with variable nodes and non-unary factors with constraint nodes. The model therefore defines a state-conditioned MRF
\begin{equation}
    p_{\EPARAM_{\NNPARAM}(\PROB,\FMP_t,t)}(\XVEC\mid \PROB,\FMP_t,t)
    \propto
    \exp\left(
        \sum_{\FACTOR\in\FACTORSET}
        \left\langle
            \EPARAM_{\NNPARAM,\FACTOR}(\PROB,\FMP_t,t),
            \SSTAT_{\FACTOR}(\XVEC_{\FACTOR})
        \right\rangle
    \right),
    \label{eq:flow_time_dependent_mrf}
\end{equation}
where the factor parameters depend on the instance, the current refinement state, and the refinement time.

This follows the approximate-inference perspective introduced in Section~\ref{sec:mrf_intro}, where LBP and NMF are used as deterministic iterative inference procedures, and the differentiable-inference viewpoint of \citet{domke_learning_2013}. With LBP, the model returns approximate unary beliefs $\widehat\MARGINAL_i(x_i)$ after a fixed number of message-passing iterations. With NMF, it returns the corresponding factorized marginals. In both cases, the endpoint score used by the rollout is
\begin{equation}
    \widehat q_{1,i}
    =
    \widehat\MARGINAL_i(x_i=1).
    \label{eq:flow_score_from_marginal}
\end{equation}
Thus, the endpoint estimate is not produced independently for each variable. It is the result of approximate inference in a factor graph whose potentials are conditioned on the current state of the refinement trajectory.

This gives the model two mechanisms for incorporating structure. First, the neural network observes the current relaxed assignment through the appended state features. Second, the MRF layer propagates local factor information through LBP or NMF before the decoder fixes variables. The flow model therefore acts as a learned refinement map over solution states, while the MRF layer preserves the constraint-aware inference mechanism introduced in the previous chapter.

\subsection{Velocity prediction by flow matching}

The first training objective follows the flow-matching viewpoint. Along the linear path in Equation~\eqref{eq:flow_probability_path}, the ideal velocity is
\begin{equation}
    \FMVELOCITY_t = \frac{d \FMP_t}{dt} = \FMPREF - \FMSOURCESTATE .
    \label{eq:flow_target_velocity}
\end{equation}
This target does not depend on $t$ for the linear path, but the model must infer it from the current state $\FMP_t$, the time $t$, and the problem instance $\PROB$.

In the implementation used here, the model predicts an endpoint estimate and the velocity is obtained from that endpoint. Given the fixed graph and the time-dependent feature collection, let
\begin{equation}
    \widehat \FMPRED_1 = \FMMODEL(\VCG, \FMFEATURES_t) \in [0,1]^n
\end{equation}
denote the predicted clean endpoint. For MRF-augmented models, this endpoint is the vector of unary beliefs defined in Equation~\eqref{eq:flow_score_from_marginal}.

The endpoint estimate defines an instantaneous velocity at time $t$ by asking which straight-line velocity would move the current point $\FMP_t$ to the predicted endpoint at time $1$:
\begin{equation}
    \widehat \FMPREDVELOCITY_t = \frac{\widehat \FMPRED_1 - \FMP_t}{1-t}.
    \label{eq:endpoint_to_velocity}
\end{equation}
In practice, this expression is evaluated only for timesteps bounded away from $t=1$ during velocity prediction, or with a small numerical safeguard in the denominator.

The instantaneous-velocity objective is therefore
\begin{equation}
    \FMLOSS(\omega) = \mathbb{E}_{(\PROB,\FMPREF),\,\FMSOURCESTATE,\,t}
    \left[
        \ell\!\left(
            \frac{\FMMODEL(\VCG,\FMFEATURES_t)-\FMP_t}{1-t},
            \FMPREF-\FMSOURCESTATE
        \right)
    \right],
    \label{eq:instant_velocity_loss}
\end{equation}
where $\ell$ is a regression loss, such as mean-squared error or $\ell_1$ loss.

At inference time, we integrate the learned dynamics by an explicit Euler scheme. Starting from $\FMP^{(0)}=\FMSOURCESTATE$, choose a grid
\begin{equation}
    0=t_0<t_1<\cdots<t_K=t_{\mathrm{end}}\leq 1,
\end{equation}
and let $\Delta t_k=t_{k+1}-t_k$. At step $k$, the model predicts an endpoint estimate
\begin{equation*}
    \widehat \FMPRED_1^{(k)}
    =
    \FMMODEL(\VCG, \FMFEATURES_{t_k}),
\end{equation*}
which is converted into a velocity
\begin{equation}
    \widehat \FMPREDVELOCITY_{t_k}
    =
    \frac{\widehat \FMPRED_1^{(k)}-\FMP^{(k)}}{1-t_k}.
\end{equation}
The relaxed assignment is then updated as
\begin{equation}
    \FMP^{(k+1)}
    =
    \FMP^{(k)} + \Delta t_k \widehat \FMPREDVELOCITY_{t_k}.
    \label{eq:flow_euler_update}
\end{equation}
The state is optionally clipped to $[0,1]^n$ before being passed to the next refinement step.

\subsection{Direct denoising along the flow path}

The second flow objective uses the same interpolation path but changes the supervision target. Instead of training the endpoint-induced velocity to match Equation~\eqref{eq:flow_target_velocity}, we train the model to predict the clean endpoint directly. The model receives the same input $\FMFEATURES_t$, containing the problem instance, the current relaxed state $\FMP_t$, and the time $t$, but the target is now the reference solution:
\begin{equation}
    \FMMODEL(\VCG, \FMFEATURES_t) \approx \FMPREF .
\end{equation}
The corresponding denoising loss is
\begin{equation}
    \FMDLOSS(\omega)
    =
    \mathbb{E}_{(\PROB,\FMPREF),\,\FMSOURCESTATE,\,t}
    \left[
        \ell\!\left(\FMMODEL(\VCG, \FMFEATURES_t), \FMPREF \right)
    \right].
    \label{eq:flow_denoising_loss}
\end{equation}
When the model outputs logits, $\ell$ is a binary cross-entropy loss. When the model outputs probabilities or MRF marginals, $\ell$ can be a mean-squared error, an $\ell_1$ loss, or the marginal cross-entropy loss used for the MRF-based models in Chapter~\ref{chapter:3_differentiable_mrf_inference_for_BILPs}.

Although the training target is different, the test-time rollout can still use the same endpoint-velocity update as in Equation~\eqref{eq:flow_euler_update}. The reason is that an endpoint predictor implicitly defines a velocity field: if the model predicts terminal state $\widehat \FMPRED_1$, then the straight-line velocity from the current state $\FMP_t$ to that endpoint is $(\widehat \FMPRED_1-\FMP_t)/(1-t)$. Thus the direct denoising model is trained as an endpoint predictor but can be unrolled as a flow model.

\section{Diffusion-Inspired Denoising Model}
\label{sec:diffusion_inspired_denoising_model}

The second refinement objective is a pseudo-diffusion denoising task. Unlike the flow models, which train on points along a source-to-solution path, the denoising model is trained on corrupted reference solutions. The goal is to reconstruct the clean assignment from the corrupted state, the timestep, and the BILP instance. We use the term diffusion-inspired because the procedure uses timestep-conditioned corruption and denoising, but the test-time sampler is a heuristic reverse denoising loop rather than an exact learned reverse Markov chain.

This section only specifies the corruption and evaluation procedure used in our experiments. The general denoising motivation was introduced in Section~\ref{sec:generative_denoising}.

\subsection{Replacement corruption process}
\label{subsec:replacement_corruption_process}

Let $\PROB$ be a BILP instance and let $y\in\{0,1\}^n$ be a reference solution. We use a discrete timestep $t\in\{0,\ldots,T-1\}$ and a signal-preservation coefficient $\bar{\alpha}_t$. Larger timesteps correspond to stronger corruption.

In the experiments, we use replacement corruption. For each variable $i$,
\begin{equation}
    y_{t,i}
    =
    \begin{cases}
        y_i, & \text{with probability } \bar{\alpha}_t,\\
        \xi_i, & \text{with probability } 1-\bar{\alpha}_t,
    \end{cases}
    \qquad
    \xi_i \sim \mathrm{Bernoulli}(1/2).
    \label{eq:diffusion_replace_corruption}
\end{equation}
The corruption acts independently across variables. Thus, the corrupted vector keeps part of the reference solution and replaces the remaining entries by unbiased binary noise.

The schedule is constructed from per-step noise levels $\beta_t$, with $\alpha_t=1-\beta_t$ and
\begin{equation}
    \bar{\alpha}_t = \prod_{\tau=0}^t \alpha_\tau .
\end{equation}
We use either a linear or cosine schedule for $\beta_t$. The implementation then corrupts each variable by keeping the clean value with probability $\bar{\alpha}_t$ and otherwise replacing it by a Bernoulli$(1/2)$ draw.

The corrupted assignment is appended as an additional variable-node feature. When timestep conditioning is enabled, the normalized timestep $t/(T-1)$ is appended as a second additional variable-node feature. Non-variable nodes receive zeros in these added feature coordinates.

\subsection{Denoising objective}
\label{subsec:diffusion_denoising_objective}

Let $D_\omega(\PROB,y_t,t)$ denote the model output. Depending on the architecture, this output may be direct variable scores or MRF-derived unary marginals. In all cases, it is interpreted as a vector of scores for the clean assignment.

During training, timesteps are sampled independently for the variables, and the model is trained to reconstruct the original solution:
\begin{equation}
    \mathcal{L}_{\mathrm{diff}}(\omega)
    =
    \mathbb{E}_{(\PROB,y)}
    \mathbb{E}_{t}
    \mathbb{E}_{y_t\sim q(\cdot\mid y,t)}
    \left[
        \ell\bigl(D_\omega(\PROB,y_t,t),y\bigr)
    \right].
    \label{eq:diffusion_denoising_loss}
\end{equation}
For binary variable-score outputs, $\ell$ is a reconstruction loss such as binary cross-entropy. For MRF-based models, the predicted scores are the unary marginals produced by LBP or NMF.

\subsection{Test-time use}
\label{subsec:diffusion_test_time_use}

At validation time, the model is evaluated as a one-shot denoiser: a clean reference solution is corrupted at one or more fixed timesteps, the model predicts clean scores, and these scores are greedily decoded. This is not an exact reverse diffusion sampler.

At test time, we evaluate two related uses. First, a corrupted reference solution is denoised either directly or through the iterative pseudo-diffusion procedure. Second, a pure random binary state can be used as the initial state. In the iterative procedure, the model is evaluated from the chosen starting timestep down to zero. At each step, the model predicts scores, these scores are converted to probabilities if necessary, and a new binary state is sampled or thresholded. No additional forward corruption is applied inside this reverse loop.

The final binary state or final model scores are then passed to the same feasibility-preserving decoders used for the flow models. This procedure should be interpreted as a learned denoising heuristic, not as an exact probabilistic reverse process.

\section{Decoding by Unrolling and Feasibility-Preserving Rounding}
\label{sec:chapter_4_unrolling_decoding}

The flow and denoising models produce relaxed states, binary states, or variable scores depending on the evaluation mode. These outputs are not guaranteed to be feasible BILP solutions. We therefore keep the same separation between prediction and decoding used earlier in the thesis: the learned model produces a score signal, and a feasibility-preserving decoder converts it into a valid assignment.

This separation is important because the intermediate refinement states may be fractional, stochastic, or constraint-violating. It also allows the experiments to isolate the effect of the refinement model itself: the final decoding mechanisms are fixed, while the score-generation process changes from one-shot prediction to learned refinement.

The decoding pipeline has three stages. We initialize a candidate state, unroll the learned flow or denoising dynamics for a prescribed number of steps, and decode the resulting state or scores with EGG, GD, or backend reinference.

\subsection{Random Bernoulli Initialization}

For the flow models, test-time refinement starts from an unstructured vector of Bernoulli parameters. Unless otherwise stated, we sample
\begin{equation}
    p^{(0)}_i \sim \mathcal{U}(0,1),
    \qquad i=1,\ldots,n.
    \label{eq:uniform_initialization}
\end{equation}
The vector $p^{(0)}\in[0,1]^n$ is not a binary assignment. Instead, each coordinate parameterizes an independent Bernoulli distribution over the corresponding decision variable. This is the sense in which the source state is a random Bernoulli initialization.

For the flow models, this vector is the source state:
\begin{equation}
    p^{(0)} = \FMSOURCESTATE .
\end{equation}
The model is then unrolled forward in time, producing a sequence of relaxed Bernoulli-parameter vectors
\begin{equation}
    p^{(0)}, p^{(1)}, \ldots, p^{(K)} \in [0,1]^n.
\end{equation}
The final vector is interpreted as variable-wise scores or Bernoulli parameters and is passed to a feasibility-preserving decoder.

For the diffusion-inspired denoising model, the test-time initialization depends on the evaluation mode. In the corrupted-reference setting, the initial state is obtained by applying the replacement corruption process of Equation~\eqref{eq:diffusion_replace_corruption} to a reference solution. In the pure-noise setting, the initial state is a binary vector sampled independently from $\mathrm{Bernoulli}(1/2)$. The denoising model is then applied from the chosen initial timestep down to zero, and the resulting state or scores are decoded.

Thus, random Bernoulli-parameter initialization is the source distribution for the flow models. The diffusion-inspired models instead test denoising from either corrupted reference assignments or pure binary noise.

\subsection{Full and Partial Unrolling}

A distinctive feature of the models in this chapter is that prediction quality depends on the number of refinement steps. Let $K$ denote the number of unrolling steps. For the flow models, we choose a time grid
\begin{equation}
    0=t_0<t_1<\cdots<t_K=t_{\mathrm{end}},
\end{equation}
and apply the learned update rule along this grid. For the diffusion-inspired model, we apply the denoiser over a sequence of timesteps, typically from a high-noise state toward a low-noise state.

When $t_{\mathrm{end}}=1$, the flow model is fully unrolled to the terminal time. We refer to this setting as \emph{full unrolling}. The resulting vector is intended to approximate the endpoint of the learned transport process. In contrast, \emph{partial unrolling} stops the dynamics earlier, with $t_{\mathrm{end}}<1$ or with fewer refinement steps. The decoder is then applied to an intermediate state.

Partial unrolling is useful for two reasons. First, it defines a runtime--quality trade-off. Fewer refinement steps reduce inference time, but may provide weaker scores. Second, the best decoded solution need not always be obtained after the longest possible rollout. In discrete optimization, a relaxed trajectory can drift toward high-confidence but structurally poor assignments, especially when the model is iterated beyond the regime in which it was most accurate. 

\subsection{Backend reinference during decimation}
\label{subsec:backend_reinference_decimation}

For MRF-based refinement models, we also evaluate a backend variant in which GD requests updated beliefs during decimation. This follows the same decimation interface as the previous chapter, but the backend is now state-conditioned.

When GD has fixed a partial assignment $S\subseteq \VCGV\times\{0,1\}$, the backend constructs an evidence-consistent source state:
\begin{equation}
    p^{(0)}_i =
    \begin{cases}
        0, & (i,0)\in S,\\
        1, & (i,1)\in S,\\
        \text{random}, & i \text{ is unfixed}.
    \end{cases}
    \label{eq:evidence_conditioned_source_state}
\end{equation}
The refinement model is then evaluated from this state, and approximate inference is run under the same evidence-clamping mechanism used previously. The resulting beliefs are returned to the GD rule.

This variant tests whether learned refinement remains useful when combined with adaptive decimation. It is more expensive than decoding a single refined score vector, because the backend may repeat refinement and inference during one GD pass.

The generic unrolling-and-decoding procedure is summarized in Algorithm~\ref{alg:unroll_decode}.

\begin{algorithm}[t]
\caption{Unrolling and feasibility-preserving decoding}
\label{alg:unroll_decode}
\DontPrintSemicolon

\KwIn{Problem instance $P$, learned refinement model $R_\omega$, number of unrolling steps $K$, decoder $\mathrm{Dec}$}
\KwOut{Feasible assignment $\hat{x}$}

Sample an initial relaxed state $p^{(0)} \sim \mathcal{U}(0,1)^n$\;
$s^{(0)} \leftarrow p^{(0)}$\;

\For{$k \leftarrow 0$ \KwTo $K-1$}{
    Compute the current timestep $t_k$\;
    Predict a refined state or update using $R_\omega(P,s^{(k)},t_k)$\;
    Update the current state to obtain $s^{(k+1)}$\;
}

Interpret $s^{(K)}$ as variable scores\;
$\hat{x} \leftarrow \mathrm{Dec}(P,s^{(K)})$\;

\Return{$\hat{x}$}\;

\end{algorithm}

\section{Main Results}
\label{sec:chapter_4_main_results}

We now report the main experimental results for the flow-based refinement models. The results are summarized in Table~\ref{tab:flow_inference_results}. We compare direct variable-score flow models, denoted DGN, with MRF-augmented flow models using either LBP or NMF inference. The suffix ``-v'' denotes models trained with the velocity-prediction objective, while the suffix ``-q'' denotes models trained to predict the denoised endpoint directly.

For each model, we report three decoding variants. First, the raw rollout sample is evaluated for feasibility before any feasibility-preserving projection is applied. This measures whether the learned refinement dynamics themselves tend to produce valid assignments. Second, the final rollout state is decoded with EGG, which uses the scores produced after the rollout but does not update them during the construction process. Third, the final rollout state is decoded with GD, which uses the same feasibility-preserving decimation procedure with score updates. For MRF-augmented models, we additionally report a backend GD variant, in which the flow-conditioned MRF signal is recomputed during decimation.

Across the experiments, the most consistent pattern is that MRF-augmented flow models produce substantially better decoding signals than direct variable-score flow models. This is especially clear on Set Cover, where the FMRF-LBP variants achieve low relative gaps and high raw-sample feasibility. The difference between velocity prediction and direct endpoint denoising is generally smaller than the difference between factorized prediction and MRF-augmented prediction, suggesting that the structured inference layer is the main source of improvement in these experiments.

\subsection{Runtime--quality trade-offs for flow decoders}

Figure~\ref{fig:flow_fix_and_solve_pareto} summarizes the runtime--quality trade-offs for the flow-based models. Each point corresponds to a model--decoder combination, with runtime on the horizontal axis and relative gap on the vertical axis. The dashed curve marks the non-dominated frontier within each dataset: points on this curve are not simultaneously worse in both runtime and solution quality than another evaluated configuration.

This figure complements Table~\ref{tab:flow_inference_results}. The table reports averaged metrics for each method, whereas the Pareto plot shows which configurations remain competitive once runtime is considered. This is important because backend GD and repeated refinement can reduce the gap, but they also add substantial computational cost.

On Set Cover, the Pareto frontier is dominated by MRF-augmented flow models, especially the LBP-based variants. Direct DGN flow models are fast but remain at larger gaps. FMRF-LBP variants move the frontier downward, showing that the additional structured inference cost can be justified when solution quality is prioritized. The NMF-based variants generally occupy an intermediate region: they improve over direct flow prediction, but are often dominated by LBP-based configurations.

On MIS, the trade-off is less favorable. MRF-augmented flow models improve over direct flow prediction, but the best low-gap points require more expensive decoding. This is consistent with the table-based results: refinement helps produce better scores, but the exclusion structure of MIS makes sequential decoding more sensitive to early ranking errors. As a result, the runtime cost of backend or more elaborate decoding is less uniformly rewarded than on Set Cover.

Overall, the Pareto plot supports the main interpretation of the flow experiments. The value of the proposed models is not only that they improve average gap, but that they shift the runtime--quality frontier for Set Cover. For MIS, the improvement is present but more constrained by the interaction between refined scores and irreversible decoding decisions.

\subsection{Set Cover Results}

The Set Cover results show a clear advantage for the MRF-augmented flow models. On SC-S, the direct DGN flow baselines obtain EGG and GD gaps above $12\%$, whereas the FMRF-LBP variants reduce the gap to approximately $2.3$--$2.4\%$ with EGG and GD. The FMRF-NMF variants also improve over DGN, although their gaps remain higher than those of FMRF-LBP. This indicates that the learned flow trajectory benefits from structured inference, and that LBP provides a stronger decoding signal than the fully factorized NMF approximation in this setting.

The raw feasibility rate also increases sharply when using MRF-augmented models. For SC-S, the DGN variants produce feasible rollout samples in only a small fraction of cases, while FMRF-LBP produces feasible samples in more than half of the test instances. This is an important observation: although feasibility is ultimately enforced by the decoder, the learned MRF-augmented dynamics already move the relaxed state toward assignments that are much more compatible with the constraints.

The same trend appears on SC-M. The DGN baselines have gaps around $14$--$16\%$, while FMRF-LBP reduces the EGG and GD gaps to roughly $2.9\%$. The feasibility of raw rollout samples is also much higher for FMRF-LBP, reaching around $70$--$80\%$ depending on the training objective. This suggests that the MRF-augmented flow model scales more favorably than the direct variable-score flow model when moving from small to medium Set Cover instances.

The comparison between velocity prediction and direct endpoint prediction is more subtle. On SC-S and SC-M, the ``-v'' and ``-q'' variants of FMRF-LBP are very close in both gap and runtime. Direct endpoint prediction is slightly better in some cases, while velocity prediction is slightly better in others. The main conclusion is therefore not that one target uniformly dominates the other, but rather that both objectives become effective once combined with an MRF-structured prediction layer.

The backend GD variant provides a different trade-off. On SC-S, recomputing the flow-conditioned MRF signal during decimation gives the best flow-model gaps, reaching around $1.3$--$1.4\%$ for FMRF-LBP. This comes at a substantially higher runtime cost. On SC-M, however, backend GD is not uniformly beneficial: the extra conditioning and recomputation increase the runtime significantly and can also worsen the average gap relative to the simpler EGG or GD projection from a single rollout. This suggests that, for Set Cover, most of the useful information may already be contained in the final rollout state, and that repeated flow-conditioned reinference must be used carefully.

\subsection{Maximum Independent Set Results}

The Maximum Independent Set results are more mixed. On MIS-S, the direct DGN flow baselines obtain gaps above $22\%$, which is comparable to the difficulty of factorized prediction observed in the previous chapter. The FMRF-LBP models improve over these baselines, especially when decoded with EGG. In particular, the direct endpoint FMRF-LBP model obtains an EGG gap around $14.3\%$, which is the best flow result reported for MIS-S in Table~\ref{tab:flow_inference_results}.

However, the behavior of GD is less favorable on MIS than on SC. For FMRF-LBP, moving from EGG to GD does not consistently improve the result, and for the NMF variants it substantially worsens the gap. This contrasts with the results of Chapter~\ref{chapter:3_differentiable_mrf_inference_for_BILPs}, where GD was often beneficial because conditioning the MRF on partial assignments provided updated marginals for the remaining variables. In the flow setting, the additional refinement and conditioning steps appear to be less stable for MIS.

This behavior is consistent with the structure of the MIS problem. MIS is governed by hard pairwise exclusion constraints: selecting one vertex immediately rules out all of its neighbors. A small error early in the construction process can therefore remove many useful candidate vertices and lead the decoder toward a poor maximal independent set. In Set Cover, by contrast, the consequences of adding a set are often less brittle, since additional selected sets can still repair uncovered elements. The MIS results therefore highlight a limitation of the refinement approach: a learned trajectory that produces useful scores on average may still interact poorly with a sequential decoder when early decisions have strong irreversible effects.

The feasibility of the raw rollout samples also shows an interesting pattern. FMRF-LBP produces feasible raw samples much more often than the DGN baselines, with feasibility rates above $50\%$ on MIS-S. Nevertheless, this does not directly translate into very low optimality gaps. Feasibility alone is therefore not enough: for MIS, the model must also rank competing feasible assignments well enough for the decoder to select a large independent set.

Overall, the MIS results show that MRF-augmented flow models can improve over factorized flow predictors, but they do not yet reproduce the strong gains obtained by MRF inference and guided decimation in Chapter~\ref{chapter:3_differentiable_mrf_inference_for_BILPs}. This suggests that, for packing problems with exclusion constraints, the interaction between refinement dynamics and sequential decoding remains a central challenge.

\subsection{Comparison with the Previous Chapter}
\label{subsec:comparison_with_previous_chapter}

Table~\ref{tab:mrf_nair_inference_results} reports the corresponding non-refinement results from Chapter~\ref{chapter:3_differentiable_mrf_inference_for_BILPs}. Comparing it with Tables~\ref{tab:flow_inference_results} and~\ref{tab:denoising_inference_results} clarifies the role of learned refinement.

On Set Cover, refinement improves the quality of one-shot decoding scores. This is most visible under EGG, which uses a fixed score vector and therefore directly tests the model output before adaptive reinference. On SC-M, the FMRF-LBP flow models obtain EGG gaps below $3\%$, improving over the corresponding non-refinement LMRF-LBP EGG result.

Under GD, however, the earlier MRF-GD models remain very competitive because they repeatedly condition the MRF on the partial assignment during decimation. Refinement therefore does not replace adaptive MRF-guided decimation. Instead, it provides a stronger score vector before decoding, while adaptive GD remains useful when its additional cost is acceptable.

On MIS-S, flow refinement improves over direct flow prediction and over some one-shot baselines, but it does not match the strongest adaptive GD results. This suggests that refinement is easier to exploit for covering problems than for packing problems with exclusion constraints.

The denoising comparison is currently restricted to Set Cover. The completed rows show the same qualitative conclusion as the flow table: LBP-based MRF refinement gives the strongest denoising signal among the evaluated variants.

\section{Discussion and Limitations}
\label{sec:chapter_4_discussion_limitations}

The completed experiments show that learned refinement is most effective when it is coupled with structured MRF inference. Direct refinement models improve over unstructured baselines in some settings, but the strongest completed results come from the MRF-augmented variants, especially those using LBP. This supports the main design principle of the chapter: the refinement model should not only update variable-wise scores, but should update a structured prediction mechanism whose approximate marginals are then used for decoding.

The difference between LBP and NMF is important. On SC-M, the FMRF-LBP flow models obtain gaps around $2.9\%$, whereas the FMRF-NMF variants remain around $6.6$--$6.7\%$ under EGG or GD. The completed denoising rows show the same ordering, with DLBP outperforming DNMF on both SC-S and SC-M. This difference is consistent with the approximate-inference discussion in Section~\ref{sec:mrf_intro}: LBP propagates factor-level information through the graph, whereas NMF uses a factorized approximation that weakens dependencies between variables.

The main problem-dependent limitation is the interaction between refinement and decoding. On Set Cover, refined scores work well with feasibility-preserving decoding because mistakes can often be repaired by selecting additional sets. On MIS, the situation is more brittle: selecting one vertex immediately excludes its neighbors, so early ranking errors can permanently remove good candidates. This helps explain why the MIS-S flow results improve over direct prediction but remain weaker than the strongest adaptive GD results from the previous chapter.

A second limitation is computational. Refinement introduces an additional test-time budget through the number of unrolling steps. Backend GD variants are even more expensive because they may repeat refinement and approximate inference during decimation. The results therefore suggest a trade-off: single-rollout refinement is attractive when a strong fixed score vector is needed, while adaptive MRF-GD remains preferable when the additional inference cost is justified.

A third limitation is experimental completeness. The denoising results currently support conclusions only for Set Cover. The MIS denoising rows are incomplete or contain placeholder values, so they should not be interpreted here. Once those experiments are complete, MIS denoising should be analyzed separately from the current flow-based MIS conclusions.

Finally, both the flow and denoising objectives rely on reference solutions. This is natural for the supervised denoising framework introduced in Section~\ref{sec:generative_denoising}, but it also limits the model to the solution patterns represented in the training data. For instances with many structurally different near-optimal solutions, future work should consider multi-solution supervision, objective-aware losses, or decoding-aware training objectives.

\section{Conclusion}
\label{sec:chapter_4_conclusion}

This chapter extended the MRF-based BILP prediction framework from one-shot inference to learned refinement dynamics. The proposed models condition their predictions on the current candidate state and a refinement time, using either flow-based objectives or replacement-corruption denoising.

The main empirical finding is that refinement is most effective when combined with structured MRF inference. On Set Cover, LBP-based refinement gives the strongest completed flow and denoising results, while direct variable-score refinement is substantially weaker. This supports the central design principle of the thesis: neural models are most useful for these BILP tasks when their predictions are mediated by structured inference and feasibility-preserving decoding.

The results also show that refinement is not a universal replacement for adaptive MRF-guided decimation. It is particularly effective as a way to improve one-shot decoding scores, whereas repeated reinference remains costly and problem-dependent. MIS highlights this limitation because hard exclusion constraints make sequential decoding sensitive to early ranking errors.

The next chapter addresses a complementary scalability issue: how to map neural features to factor potentials more efficiently. This is important because the effectiveness of the refinement models in this chapter depends on the ability to construct useful MRF parameters at each refinement step.
