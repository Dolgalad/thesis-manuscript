Interpretation

Fixing more variables changes both the residual search and the risk

The trade-off is along each curve

Increasing p leaves fewer variables to SCIP, but also commits the solver to more learned fixings.

EGG and GD define different fixing policies

EGG uses one set of scores, while GD updates scores through conditional inference as the partial assignment grows.

Runtime is not only solver time

For one-shot EGG, residual SCIP solving usually dominates. For LBP+GD, repeated inference and decoding can become a major runtime component.

Runtime split

The model forward pass is small. The practical question is whether more adaptive fixing makes the residual SCIP solve sufficiently easier.

Fix-and-solve evaluates prediction as solver assistance: the best method is the one that gives useful fixings at the right fixing ratio, not simply the fastest standalone decoder.
This slide should be read as a family of Pareto curves, not isolated method points. For each method and decoder, p varies from 0.5 to 0.9. Increasing p fixes more variables before SCIP is called. This can reduce the residual search space, but also increases the risk of committing to wrong fixings. Mention that the average runtime decomposition shows different regimes: one-shot EGG is mostly dominated by the residual SCIP solve, whereas LBP+GD can spend a large part of the total time in repeated inference and decoding.
