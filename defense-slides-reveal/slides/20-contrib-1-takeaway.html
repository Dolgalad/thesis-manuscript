Takeaway

Marginals are useful under partial information

Independent scoring score each variable separately
→
MRF inference reason over coupled assignments
1

Output dependencies matter

ILP constraints couple decision variables, so predicting each variable independently loses useful combinatorial structure.

2

Marginals give stronger decoding signals

Approximate MRF inference turns learned factor parameters into beliefs that summarize local compatibility with the remaining problem.

3

Conditioning makes decoding adaptive

Guided decimation fixes part of the solution, recomputes marginals, and uses the updated context to choose the next variables.

This is the closing slide for Contribution 1. The key message is that marginals are useful when we only have partial information. In the MRF setting, partial information comes from guided decimation: once some variables are fixed, inference recomputes beliefs for the remaining variables. This works because constraints create dependencies between output variables. The same idea will appear again in the diffusion contribution, where a corrupted solution can be interpreted as partial information about the clean solution we want to recover.
